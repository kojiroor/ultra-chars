# 要件定義書

## 1. 概要

本ドキュメントは、ウルトラマンキャラクター閲覧サイトの要件を定義する。
対象サイト: https://m-78.jp/character/

## 2. 目的

ウルトラマン公式サイトのキャラクター情報を、ユーザーが快適に閲覧できる静的サイトを提供する。

## 3. 機能要件

### 3.1 TOPページ
- サイトの説明とキャラクター一覧への導線を提供
- シンプルで分かりやすいUI

### 3.2 キャラクター一覧ページ
- 全キャラクターの一覧表示
- ページネーション機能（適切な件数で区切る）
- キャラクターのサムネイル画像、名前を表示
- キャラクター詳細ページへのリンク

### 3.3 キャラクター詳細ページ
- キャラクターの詳細情報表示
- 画像（複数の場合あり）
- 名前、説明、その他公式サイトにある情報

### 3.4 ナビゲーション
- ヘッダーによるページ遷移
- TOP ↔ 一覧 ↔ 詳細 の行き来が可能

## 4. 非機能要件

### 4.1 パフォーマンス
- 初回読み込み時間: 3秒以内
- 静的サイトとして高速なページ遷移

### 4.2 レスポンシブデザイン
- モバイル、タブレット、デスクトップに対応
- Tailwind CSSによるブレークポイント活用

### 4.3 アクセシビリティ
- セマンティックなHTML構造
- 適切なalt属性、ARIAラベル

### 4.4 ブラウザ対応
- モダンブラウザ（Chrome, Firefox, Safari, Edge）の最新2バージョン

## 5. 技術スタック

| 項目 | 技術 |
|------|------|
| 言語 | TypeScript |
| フレームワーク | React 18+ |
| ビルドツール | Vite |
| スタイリング | Tailwind CSS |
| UIコンポーネント | shadcn/ui |
| 状態管理 | Jotai (必要に応じて) |
| データ管理 | ローカルJSON (ビルド時埋め込み) |
| デプロイ | GitHub Pages |

## 6. データ構造

### 6.1 キャラクターデータ
```typescript
interface Character {
  id: string;
  name: string;
  thumbnail: string;
  images: string[];
  description: string;
  // その他、公式サイトから取得できる情報
}
```

### 6.2 データ取得方法
- ビルド時にJSONファイルとして埋め込み
- 必要に応じてスクレイピング等でデータ収集（別途検討）

## 7. 画面一覧

| 画面 | パス | 説明 |
|------|------|------|
| TOP | `/` | サイト説明、導線 |
| キャラクター一覧 | `/characters` | ページネーション付き一覧 |
| キャラクター詳細 | `/characters/:id` | 個別キャラクター情報 |

## 8. 制約事項

- データはビルド時に決定（動的取得なし）
- GitHub Pagesでのホスティング（静的サイトのみ）
- 画像は最適化して埋め込み、または外部URL参照

## 9. 今後の拡張可能性

- 検索機能
- フィルタリング機能
- キャラクターのお気に入り機能
- ダークモード対応

## 10. 用語集

| 用語 | 説明 |
|------|------|
| キャラクター | ウルトラマンシリーズに登場するキャラクター |
| ページネーション | 一覧を複数ページに分割して表示する機能 |
